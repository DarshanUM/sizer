Expected Resource Utilization shown has 3 components:
  • With no failures: Resource utilization when there are no failures in the cluster. For workloads that have replication enabled – this includes the replication overhead.
  • With local failures: Resource utilization when there are local node failures in the cluster. The number of failures assumed match the sizing parameter used for specifying the “Performance Headroom” parameter. For replicated workloads it includes the replication overhead.
  • With DR partner failure + local failures: This shows the resource utilization when DR partner cluster has failed and workloads running on that cluster have moved over. This is only applicable if the workloads being sized have replication enabled.
